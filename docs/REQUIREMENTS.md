## 中古車自動巡回システム 開発仕様書

### 1. プロジェクト概要

**目的:** 主要中古車販売・ポータルサイトを巡回し、特定の条件（N-BOX）に合致する新着情報を自動収集・通知する。
**ターゲット:**

* **車種:** ホンダ N-BOX（2020年式以降）
* **走行距離:** 5万km以下
* **巡回対象（優先順位順）:**
1. カーセンサー (CarSensor)
2. グーネット (Goo-net)
3. ネクステージ (Nextage)
4. ガリバー (Gulliver)



---

### 2. システムアーキテクチャ

「アダプターパターン」を採用し、サイトごとに異なるHTML構造を共通のデータ形式に変換します。

* **Scraper Engine:** Playwrightを用いた各サイトの巡回ロジック。
* **Data Processor:** 複数サイトから集まった情報を名寄せし、重複を排除。
* **Storage:** Google Sheets APIによるデータ保存。
* **Notifier:** LINE Notifyによる新着・値下げ通知。

### 4. サイト別巡回戦略

#### ① カーセンサー (最優先)

* **URL特徴:** 検索条件を盛り込んだ非常に長いクエリパラメータを使用。
* **抽出ポイント:** 支払総額が大きく表示されているため、これを確実に取得。
* **難易度:** 中。DOM構造は比較的安定している。
* **参照:** [カーセンサー構造定義書](./carsensor.md)


#### ② グーネット

* **URL特徴:** POST/Sessionベースの検索結果ページを使用しているため、URL直接叩きは非推奨。Playwrightでのフォーム操作が必要。
* **抽出ポイント:** 掲載台数が最大級のため、重複排除が重要。
* **難易度:** 中。カスタム要素の解析が必要。
* **参照:** [グーネット構造定義書](./goonet.md)
* **注意:** グーネットはPOST/Sessionベースの検索結果ページを使用しているため、URL直接叩きは非推奨。Playwrightでのフォーム操作が必要。
* **注意:** エリア選択はポップアップで行われるため、都道府県コードを指定してチェックする必要がある。
* **注意:** ページネーションは「次へ」ボタンをクリックして進む形式のため、最後のページまでループするロジックが必要。
* **注意:** 価格表示が整数部と小数部で分かれているため、両方を結合して数値化する必要がある。
#### ③ ネクステージ / ④ ガリバー (自社在庫)

* **URL特徴:** サイト内検索。
* **メリット:** ポータルサイトよりも詳細な車両状態（オプション装備等）がテキストで取得しやすい。
* **難易度:** 低。構造がシンプル。

---

### 5. 重複排除（名寄せ）ロジック

同じ個体が複数のサイトに掲載されるため、以下の条件が一致した場合は「同一車両」とみなして1行にまとめます。

1. **「年式」かつ「走行距離」かつ「色」が完全に一致**
2. **「販売店名」が類似、かつ「価格」が一致**

* *判定後:* 取得元サイトを「カーセンサー / グーネット」のように併記する。

---

### 6. 技術スタック・運用方針

* **言語:** TypeScript
* **ブラウザ操作:** Playwright (Chromium)
* **実行環境:** GitHub Actions（3時間おき実行）
* **セキュリティ:** credentials.json、config.json、LINEトークンを GitHub Secrets で管理。
