# 実装メモ・変更履歴

## 2026-03-12 データ品質改善

### 変更概要

スクレイピング結果の品質が不十分な項目についてスプレッドシート出力を絞り込み、
また年式フィルタリングをデータ取得後に追加した。

---

### 1. 支払総額 null の除外

**背景:** 支払総額が取得できなかった車両はスコア算出もできず、比較・判断に使えない。

**実装:** `src/index.ts` のフィルタリングステップで `totalPrice === null` の件を除外する。

```typescript
const filtered = allRaw.filter((l) => {
  if (l.totalPrice === null) return false;
  if (l.year === null || l.year < yearMin) return false;
  return true;
});
```

---

### 2. 外装色のスプレッドシート出力を廃止

**背景:** 外装色が取得できないケースが多く、スプレッドシート上で利用価値がない。

**実装:** `src/types.ts` の `SheetRow` / `SHEET_HEADERS` から `外装色` 列を削除。
`CarListing` の `color` フィールド自体は重複排除ロジック用に残す。

---

### 3. 年式フィルタリング（`src/index.ts`）

**背景:** 各サイトの検索条件（URL パラメータ等）では年式の絞り込みが完全でない場合があり、
`scraper_config.yearMin` より古い年式の車両が混入していた。

**実装:** スクレイピング後・重複排除前に `year < yearMin` の件を除外する。
`year === null`（取得不能）の車両も除外対象とする。

---

### 処理フロー（変更後）

```text
スクレイピング
  → フィルタリング（totalPrice null 除外 / year < yearMin 除外）  ← NEW
  → 重複排除
  → スプレッドシート書き込み
  → LINE 通知
```

## 2026-03-12 バリュー効率スコアの追加

スプレッドシートに「スコア」列を追加。1万円あたりの残存価値を数値化する。

$$Score = \frac{(15 - \text{走行距離[\text{万km}]}) + \max(0,\ 15 - (\text{現在年} - \text{年式}))}{\text{支払総額[\text{万円}]}}$$

- 期待寿命: 15万km
- 期待耐用年数: 15年（α=1万km/年）
- Score 0.18以上 → 割安、0.14以下 → 割高
